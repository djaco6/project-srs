## 8.1 Glossary

| Term | Definition | Used In |
|------|-----------|---------|
| **Annotation** | A computed result attached to an extracted entity, containing either a historical population lookup (city, country, continent) or a travel estimate (distance, foot/horse/cart durations). Annotations are generated by the processing pipeline and displayed in the Annotation Rail. | S1 (Scope), S3 (RQ01), S4, S6 (CL06) |
| **Annotation Rail** | The right-side panel of the dual-panel reading interface that displays population cards and travel estimate cards synchronized with the text panel. | S1 (Scope), S4 (RQ06, RQ07, RQ08) |
| **Background Job** | A unit of work executed asynchronously outside the HTTP request-response cycle. Alexandria uses GoodJob (a PostgreSQL-backed ActiveJob adapter) to process documents without blocking the user interface. | S2, S6 (CL03) |
| **Bracketing Years** | The two nearest data points (one before, one after the queried year) used to perform linear interpolation of a population estimate. Stored on each annotation for auditability. | S6 (CL06) |
| **Composition** | A UML relationship in which the child object cannot exist independently of its parent. Deleting the parent cascades deletion to all children. Used throughout the Alexandria data model (e.g., User owns Documents, Documents own Entities). | S6 |
| **Confidence Tier** | A 1--4 scale indicating the reliability of a historical population estimate: (1) census or registration data, (2) specialist demographic reconstruction, (3) national/continental aggregate from standard datasets, (4) pure model inference with order-of-magnitude uncertainty. | S6 (CL08) |
| **Entity** | A location name or date/year mention extracted from uploaded document text via pattern matching and database lookup. Each entity records its position in the text (character offsets), type (location or date), and resolution metadata. | S1 (Scope), S3 (RQ01), S4, S6 (CL05) |
| **Entity Extraction** | The process of identifying and classifying location names and date mentions in text. Alexandria's MVP uses regex-based pattern matching followed by database lookup against the urban populations table, rather than AI/NLP. | S1 (Scope), S3 (RQ01), S5 |
| **GIST Index** | A Generalized Search Tree index type in PostgreSQL, used by PostGIS for efficient spatial queries on geographic point data. | S6 (CL07) |
| **Haversine Formula** | A trigonometric formula for computing the great-circle (shortest) distance between two points on a sphere given their latitude and longitude. Alexandria uses this to calculate straight-line distances between co-mentioned locations, before applying a terrain multiplier. | S1 (Scope), S5, S6 (CL06) |
| **Linear Interpolation** | A method for estimating a value between two known data points by assuming a constant rate of change. Used to estimate population at a queried year when the database contains data points for surrounding years but not the exact year. | S1 (Scope), S6 (CL06) |
| **Map Strip** | The collapsible, full-width map panel at the top of the reading interface that renders detected locations as markers and draws distance lines between co-mentioned location pairs using Leaflet.js. | S1 (Scope), S4 (RQ03, RQ04, RQ05) |
| **OCR** | Optical Character Recognition -- the conversion of image-based text (photographs, scanned documents) into machine-readable character data. Alexandria supports image uploads via the RTesseract library. | S1 (Scope), S3 (RQ02), S5 |
| **Pattern Matching (Regex)** | The use of regular expressions to identify structured text patterns. Alexandria extracts years via `\b(\d{3,4})\b` and location candidates via capitalized word sequences, then validates candidates against the database. | S1 (Scope), S5, S6 (CL05) |
| **PostGIS** | A spatial database extension for PostgreSQL that adds support for geographic objects, spatial indexing (GIST), and distance functions. Enables efficient location-based queries on the urban populations dataset. | S2, S6 (CL07) |
| **Seed Data** | Pre-loaded reference datasets imported into the database during initial setup. Includes the CIESIN historical urban populations (CL07), Maddison/HYDE regional populations (CL08), and travel speed constants (CL09). These tables are read-only during normal operation. | S2, S5, S6 (CL07, CL08, CL09) |
| **Terrain Multiplier** | A scaling factor (default 1.20x) applied to straight-line Haversine distance to approximate actual overland travel distance, accounting for roads, terrain, and routing. Ranges from 1.10x (flat, well-roaded) to 2.00x (mountainous). | S1 (Scope), S5, S6 (CL06) |
| **Trigram Index** | A PostgreSQL index using the `pg_trgm` extension that enables efficient fuzzy string matching. Used on city names to support approximate location search even with minor spelling variations. | S6 (CL07) |
| **Wireflow** | A combination of wireframe (static UI layout) and flowchart showing how a user navigates between screens and states in response to actions. Each user story in Section 4 includes a wireflow illustrating the happy-path interaction. | S4 |
